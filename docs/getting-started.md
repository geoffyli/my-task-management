# Getting Started

Developer onboarding guide — from clone to running app.

## Prerequisites

- **Bun** (latest) — [bun.sh](https://bun.sh)
- **Notion Integration** — API key from [notion.so/my-integrations](https://www.notion.so/my-integrations)
  - The integration must have access to the tasks, projects, and areas databases

## Setup

### 1. Clone and Install

```bash
git clone <repo-url>
cd task-management-analytics
bun install
```

### 2. Configure Environment

```bash
cp .env.example .env
```

Edit `.env`:

```bash
NOTION_API_KEY=ntn_your_key_here
DEV=true
# TOKEN not needed in dev mode
```

Alternative: store the API key in a file:
```bash
mkdir -p ~/.config/notion
echo "ntn_your_key_here" > ~/.config/notion/api_key
```

### 3. Run Development Mode

```bash
bun run dev
```

This starts:
- **Backend** on `http://localhost:3456` (Hono server with file watching)
- **Frontend** on `http://localhost:5173` (Vite dev server with HMR)

The Vite dev server proxies `/api/*` requests to the backend automatically.

### 4. First Boot

On first run with an empty database:
1. Server creates `./data/analytics.db` automatically
2. Full sync runs (fetches all pages from Notion — takes 2-5 seconds)
3. Health check (`/healthz`) returns 503 until sync completes, then 200

### 5. Access the App

Open `http://localhost:5173`

In dev mode (with `DEV=true`), authentication is bypassed — you're logged in automatically.

## Development Commands

| Command | Description |
|---------|-------------|
| `bun run dev` | Start both backend and frontend (concurrent) |
| `bun run dev:server` | Backend only (port 3456, file watching) |
| `bun run dev:client` | Frontend only (port 5173, HMR) |
| `bun run build` | Build frontend for production (`dist/`) |
| `bun run preview` | Preview production build via Vite |
| `bun run start` | Production server (serves `dist/` + API) |

## Production Build

```bash
bun run build     # Compile frontend to dist/
bun run start     # Serve everything from one process
```

Or use Docker — see [[docker-deployment]].

## Project Structure

```
├── src/           # React frontend (pages, components, hooks, metrics)
├── server/        # Bun + Hono backend (API, database, sync)
├── public/        # Static assets (fonts)
├── data/          # SQLite database (auto-created)
├── dist/          # Built frontend (generated by vite build)
└── docs/          # This documentation
```

See [[architecture-overview]] for detailed module boundaries.

## Key Concepts

- **Sync**: Data flows one-way from Notion → SQLite. The app is read-only with respect to task data.
- **Three sync layers**: Full sync (startup), reconciliation (every 15 min), webhooks (real-time). See [[sync-overview]].
- **Client-side metrics**: All analytics computed in the browser from raw data. See [[metrics-functions]].
- **Design**: Linear-inspired dark theme. See [[design-system-moc]].

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "NOTION_API_KEY not found" | Set in `.env` or create `~/.config/notion/api_key` file |
| Empty dashboard | Check sync status in Settings page; ensure Notion integration has database access |
| Auth errors in production | Ensure `TOKEN` env var is set; generate with `openssl rand -hex 32` |
| Port conflict | Change `PORT` in `.env` (backend) or Vite config (frontend) |
| Database locked | WAL mode handles most cases; check for zombie processes holding the file |
